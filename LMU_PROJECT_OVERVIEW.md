# LMU Telemetry Lab 專案概覽

> 最後更新：2026-04-14（v1.2.0）

這份文件紀錄了 LMU Telemetry Lab 的核心功能特色以及開發過程中的關鍵里程碑。

## 🚀 核心功能亮點

### 1. 極致數據引擎 (DuckDB + FastAPI)
- **高性能分析**：利用 **DuckDB** 的列式存儲與向量化查詢技術，實現海量遙測數據的瞬時檢索。
- **低延遲通訊**：前端 React 與後端 **FastAPI** 透過高效的 JSON/Binary 協議進行通訊，動態圖表與視域切換零延遲。

### 2. 次世代視覺化賽道圖 (Next-Gen TrackMap System)
- **2D 賽道地圖**：支援毫米級精度的 GPS 投影，準確繪製賽車走線，附帶油門/煞車/滑行熱圖。
- **3D 立體賽道圖 (TrackMap3D)**：基於 Three.js / React Three Fiber 的全 3D 渲染引擎，支援高度曲線與跑線疊加。
- **智慧視角切換**：具備 **Static (固定)**、**Follow (跟隨)** 與 **Heading Up (車頭朝上自旋)** 三種專業模式。
- **全視域 MiniMap**：在主視圖底部提供縮小版全場圖，即時掌握整體動態。

### 3. F1 專業級分析介面
- **Telemetry Sync**：多頻道遙測圖表（油門、煞車、速度、RPM）全局游標同步，支援縮放。
- **Delta-T 動態分析**：實時計算與參考單圈 (Reference Lap) 之間的時間盈虧 (Delta)，並支援跨 Session 比較。
- **擬真硬體模擬**：內建數位方向盤 (Steering Wheel View，支援自訂方向盤皮膚)、輪胎溫度/壓力熱圖與油耗監測器。

### 4. 互動式 Telemetry HUD
- **可拖拽/縮放覆蓋層 (CompactTelemetryOverlay)**：在 3D 地圖上顯示當前與參考圈的即時遙測數據。
- **雙擊解鎖編輯模式**：使用者可自由調整 HUD 位置與大小，支援百分比定位確保跨解析度一致。
- **邊界保護**：整合 ResizeObserver，HUD 隨視窗縮放自動校準，防止 UI 脫域。
- **一鍵重置**：全局 Reset HUD 按鈕還原標準佈局。

### 5. 3D Ghost Car 與雙圈對比
- **幽靈車 (Ghost Car)**：在 3D 場景中同時渲染主單圈與參考單圈的位置，實現空間位置的直觀對位。
- **行車線熱圖 (Heatmap)**：3D 軌跡頂點著色與 2D 版完全同步（油門/煞車/滑行色彩映射）。
- **跨 Session 一致性**：所有數據強制同步至當前工作階段的地理中心，解決跨 Session 渲染位移問題。

### 6. 頂級視覺體驗 (Glassmorphism UI)
- **玻璃擬態設計**：完美結合 Backdrop Blur 與半透明元素，打造極具科技感的專業儀表板。
- **全局 Tooltip 系統**：自定義玻璃質感提示框，完整取代瀏覽器原生樣式。
- **多帳號 / 個人檔案**：支援多個使用者帳號，各自獨立的數據存取與自訂頭像。

## 📝 開發歷程紀錄 (Project Milestones)

### 第一階段：基礎架構 (Core Foundation)
- 建立 DuckDB Schema 並開發 LMU Telemetry 二進制格式的解析算法。
- 實現單圈自動切割與有效圈 (Valid Lap) 偵測邏輯。

### 第二階段：時效連動 (Temporal & Spatial Engine)
- 完成 `GPS -> SVG/Path` 的投影引擎，支持賽道走線可視化。
- 開發跨組件的遙測同步系統，達成 1ms 級別的游標對齊。

### 第三階段：設計革命 (Visual Overhaul)
- 視覺風格由 Standard 轉向 **Glassmorphism (玻璃擬態)**。
- 優化大尺寸、高解析度螢幕下的 Resizer 與 Side-bar 動作邏輯。

### 第四階段：數據對比 (Cross-Session Analytics)
- 成功導入「跨賽事參考瀏覽器」，允許玩家從不同的 DuckDB 數據源抓取參考單圈。
- 解決不同車型 (GT/LMH) 之間的數據歸一化比對。

### 第五階段：3D 渲染引擎 (3D Telemetry Lab)
- 基於 Three.js 的 3D 賽道渲染從 Alpha 進化至生產可用 (v1.0.5 → v1.2.0)。
- 實現 3D Ghost Car、行車線熱圖、高度曲線與跨 Session 渲染一致性。
- 航段感知數據融合 (Stint-Aware Fusion)，解決長期存在的 Lap 0 消失問題。

### 第六階段：細節與閉環 (v1.2.0 Polish)
- **互動式 HUD**：可拖拽、縮放、持久化配置的 Compact Telemetry Overlay。
- **Self-Healing Playback**：自動偵測並修復非零起始點數據（COTA 等特殊賽道）。
- **座標歸一化**：Unified Origin 機制確保所有跨 Session 數據在空間上精準對位。
- **部署閉環**：Electron 打包、版本管理、GitHub 發布流程建立完成。
